# Team Scheduler 需求系統 — 測試流程

系統網址:**https://transcend-design.web.app**
測試前請先用 Chrome 開啟網址,按 `Cmd+Shift+R`(Windows:`Ctrl+F5`)強制重新整理一次。

---

## 🟢 Planner 測試(約 10 分鐘)

### P1. 登入
- [ ] 用你的 Google 帳號登入(主管已把你加入白名單)
- [ ] 登入後左下角顯示你的名字與「Planner」
- [ ] 左側選單只看到:提交需求、我的需求、需求總表

### P2. 提交需求
- [ ] 點「📝 提交需求」,不填任何欄位直接送出 → 應出現紅字提示必填
- [ ] 完整填一筆:勾急件、選地區、專案名稱(格式 `區域_案名_類型`)、勾 2 個稿件類型、選交期、填需求簡述
- [ ] 上傳附件:選一個 PPT 或 PDF(<10MB)→ 送出 → 顯示「需求已送出」並跳到我的需求
- [ ] 試上傳一個 >10MB 的檔案 → 應被擋下並提示

### P3. 我的需求
- [ ] 剛送的需求顯示「待審核」,並標示「可編輯」
- [ ] 點該筆 → 彈出完整內容(附件可點開預覽)
- [ ] 點「✎ 編輯需求」→ 改內容、加/刪附件 → 儲存 → 內容有更新
- [ ] (等主管核准後)狀態變「已發稿」,點開可看到指派設計師與注意事項;此時**不能**再編輯

### P4. 需求總表
- [ ] 只看得到**自己負責區域**的需求(看不到其他區域)
- [ ] 點任一列 → 彈出完整內容
- [ ] 附件點了直接在網頁預覽(PPT/PDF/圖片)
- [ ] 試篩選(設計師/狀態)與排序(交期/提交日)
- [ ] 對一筆「設計稿確認中」的需求按「✓ 結案」→ 移到下方「已結案」表格(淡色)

### P5. Email
- [ ] 需求被核准時:收到 CC 信(收件人是設計師)
- [ ] 需求被駁回時:收到駁回通知信(含原因)
- [ ] ⚠️ 沒收到請先檢查**垃圾信件匣**(寄件者為 tselvis814@gmail.com)

## 🔵 設計師測試(約 10 分鐘)

### D1. 登入
- [ ] 用你的 **Gmail**(主管登記的那個)登入
- [ ] 左下角顯示你的名字與「設計師」
- [ ] 左側選單看到:需求總表 + 排程管理各頁(甘特圖等)

### D2. 接收任務
- [ ] 主管指派需求給你後,收到通知信(寄到**公司信箱**),信中有專案資訊、附件連結、「前往需求總表」按鈕
- [ ] 開「🗂️ 需求總表」→ 只看得到**指派給自己**的需求

### D3. 處理任務
- [ ] 點任一列 → 看完整需求內容、附件預覽、主管的注意事項(📌)
- [ ] 用「動作」欄下拉把狀態改成「設計中」→ 狀態即時更新
- [ ] 再改「設計稿確認中」→ 再改「已結案」→ 該筆移到下方已結案表格
- [ ] 確認你**改不了**需求的內容(標題、交期等只有主管能改)

### D4. 排程功能(原有)
- [ ] 甘特圖、日曆、專案管理、休假預排等原有頁面正常顯示

---

## 🟣 主管測試(Elvis / Renee)

### M1. 審核
- [ ] 「⚖️ 需求審核」看到待審核清單(含附件預覽)
- [ ] 核准:選 1 位以上設計師、填注意事項、改交期 → 核准並指派
- [ ] 設計師收到信(To)、planner + 主管收到 CC
- [ ] 駁回另一筆(填原因)→ planner 收到駁回信
- [ ] 對已發稿需求「✎ 編輯」新增一位設計師 → **新增的那位**收到(新增指派)信

### M2. 總覽與儀表板
- [ ] 需求總表看到**全部**需求;篩選、排序正常
- [ ] 「📈 設計師儀表板」:每位設計師的 loading 等級與時間軸色塊正確;點色塊開詳情

### M3. 系統管理
- [ ] 使用者管理:新增/編輯/停用使用者;planner 可設定負責區域
- [ ] 權限設定:取消設計師某頁勾選 → 該設計師重整後看不到那頁(測完記得改回來)
- [ ] 用一個**不在白名單**的 Google 帳號登入 → 顯示「帳號尚未開通」

## 🐛 回報問題格式

發現問題請提供:
1. 你的角色(planner / 設計師 / 主管)
2. 在哪一頁、做了什麼操作
3. 預期看到什麼 vs 實際看到什麼
4. 截圖(有紅字錯誤訊息的話一定要拍到)
